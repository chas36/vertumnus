# Vertumnus

Десктопный конвертер видео в `MP4` для старых проекторов и других устройств с ограниченной поддержкой форматов.

Приложение написано на `Python + PySide6`, использует `FFmpeg/ffprobe`, поддерживает очередь файлов, выбор качества, аудиодорожек и субтитров, а также базовый и расширенный режим интерфейса.

## Что умеет

- Конвертировать видео разных форматов в `MP4 (H.264 + AAC)`
- Работать с очередью файлов и пакетной конвертацией
- Поддерживать drag-and-drop и добавление через диалог
- Сохранять настройки и очередь между запусками
- Переключаться между базовым и расширенным интерфейсом
- Выбирать профиль `Для проектора` или `Кастомный`
- Выбирать качество видео: `Оригинал / 1080p / 720p / 480p`
- Оставлять одну нужную аудиодорожку
- Оставлять одну дорожку субтитров или отключать субтитры
- Показывать историю конвертаций и детали ошибок
- Переключаться между тёмной и светлой темой

## Для чего проект

Основной сценарий: быстро подготовить видео для старых проекторов, которые плохо работают с современными контейнерами, кодеками, несколькими аудиодорожками и переключаемыми субтитрами.

Профиль `Для проектора` делает совместимый выходной файл без необходимости вручную настраивать FFmpeg.

## Быстрый старт

### macOS

Нужны:

- `Python 3.11`
- `ffmpeg`
- `ffprobe`

Установка `ffmpeg`:

```bash
brew install ffmpeg
```

Запуск приложения:

```bash
git clone https://github.com/chas36/vertumnus.git
cd vertumnus
./run_mac.sh
```

### Windows

Сборка описана в [WINDOWS_BUILD.md](WINDOWS_BUILD.md).

Коротко:

1. Установить `Python 3.11`
2. Положить `ffmpeg.exe` и `ffprobe.exe` в `assets/ffmpeg/`
3. Выполнить `build_windows.ps1`

Для GitHub Actions это не требуется: workflow сам скачивает FFmpeg во время CI-сборки.

## Зависимости

Python-пакеты:

```bash
pip install -r requirements.txt
```

Системные зависимости:

- `ffmpeg`
- `ffprobe`

Для локального запуска они должны быть доступны в `PATH` или лежать в `assets/ffmpeg`.

## Структура проекта

```text
.
├── core/        # FFmpeg, ffprobe, воркер, история, состояние очереди
├── models/      # Dataclass-модели настроек, файлов и потоков
├── tests/       # Тесты конвертера, probe и состояния очереди
├── ui/          # Главное окно, список файлов, панель настроек, темы
├── main.py      # Точка входа
├── run_mac.sh   # Launcher для macOS
└── build.spec   # Сборка через PyInstaller
```

## Текущие возможности интерфейса

- Базовый режим:
  - добавить файлы
  - выбрать профиль
  - запустить конвертацию
- Расширенный режим:
  - выбрать папку вывода
  - настроить качество, FPS и битрейты
  - выбрать аудиодорожку
  - выбрать субтитры
  - управлять темой интерфейса

## Ограничения

- Сборка `Windows .exe` должна выполняться на Windows
- Image-based субтитры вроде `PGS` не встраиваются в `MP4`
- Лицензия проекта пока не выбрана и не добавлена в репозиторий

## Разработка

Запуск тестов:

```bash
python -m unittest discover -s tests -v
```

Проверка компиляции:

```bash
python -m compileall main.py core models ui tests
```

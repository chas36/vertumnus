## Цели и требования

### Проблема
Старые проекторы поддерживают только формат MP4 (H.264 + AAC).
Существующие конвертеры — устаревшие, с ошибками или платные.

### Цель
Простое, быстрое, бесплатное приложение для конвертации **любого видео → MP4**,
которое работает без установки дополнительного ПО.

### Функциональные требования

| #  | Требование                                              | Приоритет |
|----|---------------------------------------------------------|-----------|
| F1 | Конвертация видео любого формата в MP4 (H.264 + AAC)   | 🔴 Must   |
| F2 | Выбор файлов через диалог и drag & drop                 | 🔴 Must   |
| F3 | Очередь и пакетная конвертация нескольких файлов        | 🔴 Must   |
| F4 | Прогресс-бар в реальном времени для каждого файла       | 🔴 Must   |
| F5 | Выбор папки для сохранения результата                   | 🔴 Must   |
| F6 | Профиль "для проектора" — один клик, готовые настройки  | 🟡 Should |
| F7 | Выбор качества: разрешение, битрейт, FPS                | 🟡 Should |
| F8 | Предпросмотр метаданных файла (длит., кодек, размер)    | 🟡 Should |
| F9 | История конвертаций                                     | 🟢 Nice   |
| F10| Уведомление по завершении всей очереди                  | 🟢 Nice   |

### Нефункциональные требования

- Работает на Windows 10/11 без установки Python и FFmpeg
- Один `.exe` файл (или папка с `.exe`)
- Интерфейс на русском языке
- Не замораживает UI во время конвертации

## Архитектура

### Поток данных

```
Пользователь
    │
    ▼
[ MainWindow ]
    │  добавляет файлы
    ▼
[ FileListWidget ]  ←──────────────────────────────┐
    │  создаёт FileItem модели                      │
    ▼                                               │
[ ConversionWorker : QThread ]                      │
    │  читает FileItem                              │  emit сигналы:
    │  формирует команду FFmpeg                     │  progress_updated(int)
    │  запускает subprocess                         │  file_done(FileItem)
    │  парсит stderr → прогресс %                   │  all_done()
    └──────────────────────────────────────────────►┘
```

### Ключевые классы

```python
# models/file_item.py
@dataclass
class FileItem:
    path: Path
    status: Literal["pending", "converting", "done", "error"]
    progress: int = 0
    duration: float = 0.0
    error_message: str = ""

# models/conversion_settings.py
@dataclass
class ConversionSettings:
    output_dir: Path
    resolution: str = "original"   # "original" | "1920x1080" | "1280x720" | ...
    fps: int = 0                    # 0 = original
    video_bitrate: str = "auto"
    audio_bitrate: str = "128k"
    preset: str = "medium"          # FFmpeg preset: ultrafast → veryslow
    profile: str = "projector"      # "projector" | "custom"

# core/worker.py
class ConversionWorker(QThread):
    progress_updated = Signal(str, int)   # (file_path, percent)
    file_done = Signal(str, bool, str)    # (file_path, success, error_msg)
    all_done = Signal()
```

### Шаг 2 — Ядро конвертации
**Цель:** реально конвертировать файлы, не замораживая интерфейс.

- [ ] `probe.py` — запуск `ffprobe -v json`, парсинг длительности и кодека
- [ ] `converter.py` — формирование команды FFmpeg:
  ```
  ffmpeg -i input.avi -c:v libx264 -c:a aac -movflags +faststart output.mp4
  ```
- [ ] Парсинг прогресса из stderr FFmpeg:
  - FFmpeg пишет `time=HH:MM:SS.ms` → считаем `time / duration * 100`
- [ ] `ConversionWorker(QThread)` — обрабатывает очередь файлов по одному
- [ ] Сигналы: `progress_updated`, `file_done`, `all_done`
- [ ] Подключение сигналов к `FileItemWidget` (обновление прогресс-бара)
- [ ] Кнопка "Старт" запускает воркер, "Отмена" — останавливает

**Результат:** файлы реально конвертируются, прогресс виден в реальном времени.

## Детали реализации

### Парсинг прогресса FFmpeg

FFmpeg пишет в stderr строки вида:
```
frame=  120 fps= 24 q=28.0 size=    1024kB time=00:00:05.00 bitrate=1677.7kbits/s speed=1.0x
```

Алгоритм:
1. Запустить FFmpeg с флагом `-progress pipe:1` для структурированного вывода
2. Читать stdout построчно в отдельном потоке
3. Искать строку `out_time_ms=<microseconds>`
4. `progress = out_time_ms / (duration * 1_000_000) * 100`

### Совместимый профиль "Для проектора"

```
ffmpeg -i input.avi \
  -c:v libx264 \          # видеокодек H.264
  -profile:v baseline \   # максимальная совместимость
  -level 3.0 \            # для старых устройств
  -c:a aac \              # аудиокодек AAC
  -b:a 128k \             # битрейт аудио
  -movflags +faststart \  # быстрый старт воспроизведения
  -pix_fmt yuv420p \      # формат пикселей, совместимый с любым плеером
  output.mp4
```

### Обработка ошибок

| Ситуация                        | Поведение                                              |
|---------------------------------|--------------------------------------------------------|
| Файл уже существует             | Спросить: перезаписать / переименовать / пропустить    |
| FFmpeg не найден                | Показать ошибку с инструкцией                          |
| Входной файл повреждён          | Пометить как ❌, показать вывод FFmpeg                 |
| Мало места на диске             | Проверить перед стартом, предупредить                  |
| Пользователь нажал "Отмена"     | Завершить процесс FFmpeg, удалить неполный файл        |

## Сборка и дистрибуция

### Зависимости (`requirements.txt`)

```
PySide6>=6.6.0
pyinstaller>=6.0.0
```

> FFmpeg и ffprobe — бинарники, скачиваются отдельно с ffmpeg.org и кладутся в `assets/ffmpeg/`

### Команды разработки

```bash
# Создать виртуальное окружение
python -m venv venv
venv\Scripts\activate

# Установить зависимости
pip install -r requirements.txt

# Запустить приложение
python main.py

# Собрать .exe
pyinstaller build.spec
```

### Конфигурация PyInstaller (`build.spec`)

```python
a = Analysis(
    ['main.py'],
    datas=[
        ('assets/ffmpeg/ffmpeg.exe', 'assets/ffmpeg'),
        ('assets/ffmpeg/ffprobe.exe', 'assets/ffmpeg'),
        ('ui/styles.qss', 'ui'),
        ('assets/icons', 'assets/icons'),
    ],
    ...
)
exe = EXE(a, name='MP4Converter', icon='assets/icons/app.ico', console=False)
```

## Известные риски

| Риск                                         | Вероятность | Митигация                                              |
|----------------------------------------------|-------------|--------------------------------------------------------|
| FFmpeg падает на редком формате              | Средняя     | Показывать полный вывод ошибки пользователю            |
| PyInstaller + PySide6 = большой .exe (>80MB) | Высокая     | Нормально для десктопа; можно использовать --onedir    |
| Прогресс не парсится для некоторых файлов    | Низкая      | Fallback: крутить spinner без точного процента         |
| Пользователь не знает куда сохранился файл  | Средняя     | Уведомление + кнопка "Открыть папку"                   |
